# User Acceptance Test Cases - ระบบจัดการคลินิก

## ข้อมูลทั่วไป
- **ระบบ**: ระบบจัดการคลินิก (Clinic Management System)
- **ประเภทการทดสอบ**: User Acceptance Testing (UAT)
- **วัตถุประสงค์**: ทดสอบความพึงพอใจของผู้ใช้งานจริงและความถูกต้องตามความต้องการ
- **วันที่สร้าง**: $(date)
- **เวอร์ชัน**: 1.0

---

## Test Cases

### UAT-ID: UAT-001
**บทบาท/Use Case**: เจ้าหน้าที่: ลงทะเบียน/ค้นหาผู้ป่วย
**ขั้นตอน**:
1. ลงทะเบียนผู้ป่วยใหม่
2. ค้นหาด้วยชื่อ/เลขบัตร

**เกณฑ์ยอมรับ**: สร้างข้อมูลสำเร็จ, ค้นหาเจอภายใน 2-5 วินาที

**ข้อมูลทดสอบ**:
```json
{
  "newPatient": {
    "first_name": "สมหญิง",
    "last_name": "รักดี",
    "national_id": "2345678901234",
    "phone": "0823456789",
    "email": "somying@example.com",
    "date_of_birth": "1985-05-15",
    "gender": "female",
    "address": "123 ถนนสุขุมวิท กรุงเทพฯ 10110"
  },
  "searchCriteria": [
    {
      "type": "name",
      "value": "สมหญิง รักดี"
    },
    {
      "type": "national_id", 
      "value": "2345678901234"
    },
    {
      "type": "phone",
      "value": "0823456789"
    }
  ],
  "performanceRequirement": {
    "searchTime": "≤ 5 seconds",
    "dataAccuracy": "100%"
  }
}
```

**การตรวจสอบ**:
- [ ] ข้อมูลผู้ป่วยถูกบันทึกครบถ้วนและถูกต้อง
- [ ] ระบบแสดงข้อความยืนยันการบันทึกสำเร็จ
- [ ] ค้นหาด้วยชื่อเจอผลลัพธ์ภายใน 5 วินาที
- [ ] ค้นหาด้วยเลขบัตรประชาชนเจอผลลัพธ์ภายใน 3 วินาที
- [ ] ค้นหาด้วยเบอร์โทรศัพท์เจอผลลัพธ์ภายใน 3 วินาที
- [ ] ข้อมูลที่แสดงตรงกับที่บันทึกไว้

### UAT-ID: UAT-002
**บทบาท/Use Case**: แพทย์: บันทึกเวชระเบียน/สั่งยา
**ขั้นตอน**:
1. เปิดประวัติ
2. กรอกอาการ/วินิจฉัย
3. เลือกยา
4. บันทึก

**เกณฑ์ยอมรับ**: บันทึกสำเร็จ, แสดงใบสั่งยา, ไม่มีฟิลด์บังคับขาดหาย

**ข้อมูลทดสอบ**:
```json
{
  "patientId": "patient_123",
  "medicalRecord": {
    "chief_complaint": "ปวดหัวมา 2 วัน",
    "symptoms": "ปวดหัวข้างเดียว, คลื่นไส้, แพ้แสง",
    "diagnosis": "ไมเกรน",
    "treatment_plan": "พักผ่อน, หลีกเลี่ยงแสงจ้า, ทานยาแก้ปวด",
    "follow_up": "ติดตามอาการใน 1 สัปดาห์"
  },
  "prescription": [
    {
      "medication": "Paracetamol 500mg",
      "quantity": 20,
      "dosage": "1-2 เม็ด ทุก 4-6 ชั่วโมง",
      "duration": "3 วัน"
    },
    {
      "medication": "Ibuprofen 400mg",
      "quantity": 10,
      "dosage": "1 เม็ด หลังอาหาร",
      "duration": "2 วัน"
    }
  ],
  "requiredFields": [
    "chief_complaint",
    "diagnosis", 
    "treatment_plan"
  ]
}
```

**การตรวจสอบ**:
- [ ] ระบบแสดงข้อมูลผู้ป่วยครบถ้วน
- [ ] ฟิลด์บังคับทั้งหมดมีข้อมูล
- [ ] ระบบตรวจสอบความถูกต้องของข้อมูลก่อนบันทึก
- [ ] เวชระเบียนถูกบันทึกสำเร็จ
- [ ] ใบสั่งยาถูกสร้างและแสดงผล
- [ ] ข้อมูลในใบสั่งยาตรงกับที่เลือก
- [ ] ระบบแสดงข้อความยืนยันการบันทึก

### UAT-ID: UAT-003
**บทบาท/Use Case**: เภสัชกร: จ่ายยา/ตัดสต๊อก
**ขั้นตอน**:
1. เปิดใบสั่ง
2. ตรวจคงเหลือ
3. ยืนยันจ่าย

**เกณฑ์ยอมรับ**: ตัดสต๊อกถูกต้อง, แจ้งเตือนสต๊อกต่ำ/หมดอายุ

**ข้อมูลทดสอบ**:
```json
{
  "prescriptionId": "pres_456",
  "prescriptionItems": [
    {
      "medication": "Paracetamol 500mg",
      "prescribedQuantity": 20,
      "currentStock": 100,
      "expiryDate": "2025-12-31"
    },
    {
      "medication": "Ibuprofen 400mg", 
      "prescribedQuantity": 10,
      "currentStock": 5,
      "expiryDate": "2024-06-30"
    }
  ],
  "stockAlerts": {
    "lowStock": "Ibuprofen 400mg (เหลือ 5 ชิ้น)",
    "expiryWarning": "Ibuprofen 400mg (หมดอายุ 2024-06-30)"
  }
}
```

**การตรวจสอบ**:
- [ ] ระบบแสดงใบสั่งยาครบถ้วน
- [ ] แสดงจำนวนสต๊อกปัจจุบันของแต่ละยา
- [ ] แสดงวันที่หมดอายุของยา
- [ ] แจ้งเตือนสต๊อกต่ำสำหรับ Ibuprofen
- [ ] แจ้งเตือนยาใกล้หมดอายุ
- [ ] Paracetamol ถูกจ่ายได้ปกติ (สต๊อกเพียงพอ)
- [ ] Ibuprofen ถูกจ่ายได้ (แต่มีเตือน)
- [ ] สต๊อกถูกตัดตามจำนวนที่จ่าย
- [ ] ระบบบันทึกการจ่ายยา

### UAT-ID: UAT-004
**บทบาท/Use Case**: เจ้าหน้าที่การเงิน: ออกใบเสร็จ/รับชำระ
**ขั้นตอน**:
1. สร้างใบแจ้งหนี้
2. รับชำระ (เงินสด/QR)

**เกณฑ์ยอมรับ**: ยอดรวมถูกต้อง, ออกใบเสร็จสำเร็จ, พิมพ์/ส่งดิจิทัลได้

**ข้อมูลทดสอบ**:
```json
{
  "invoiceData": {
    "patientId": "patient_123",
    "items": [
      {
        "description": "ค่ารักษา",
        "amount": 500,
        "quantity": 1
      },
      {
        "description": "ค่ายา Paracetamol",
        "amount": 100,
        "quantity": 20
      },
      {
        "description": "ค่ายา Ibuprofen", 
        "amount": 200,
        "quantity": 10
      }
    ],
    "discount": 50,
    "taxRate": 7
  },
  "payment": {
    "method": "cash",
    "amount": 800,
    "change": 0
  },
  "expectedCalculation": {
    "subtotal": 800,
    "discount": 50,
    "tax": 52.5,
    "total": 802.5
  }
}
```

**การตรวจสอบ**:
- [ ] ระบบคำนวณยอดรวมถูกต้อง
- [ ] แสดงรายละเอียดการคำนวณ (subtotal, discount, tax, total)
- [ ] รับการชำระเงินได้
- [ ] ใบเสร็จถูกสร้างและแสดงผล
- [ ] ข้อมูลในใบเสร็จครบถ้วนและถูกต้อง
- [ ] สามารถพิมพ์ใบเสร็จได้
- [ ] สามารถส่งใบเสร็จทางอีเมลได้
- [ ] ระบบบันทึกการชำระเงิน

### UAT-ID: UAT-005
**บทบาท/Use Case**: Admin: จัดการผู้ใช้/บทบาท
**ขั้นตอน**:
1. เพิ่ม/แก้ไข/ระงับบัญชี
2. กำหนดบทบาท

**เกณฑ์ยอมรับ**: สิทธิ์มีผลทันที, ผู้ใช้ถูกจำกัดตามบทบาท

**ข้อมูลทดสอบ**:
```json
{
  "newUser": {
    "username": "newstaff01",
    "email": "newstaff@clinic.com",
    "password": "SecurePass123",
    "role": "staff",
    "status": "active"
  },
  "rolePermissions": {
    "admin": ["all_access"],
    "doctor": ["medical_records", "prescriptions", "patient_info"],
    "pharmacist": ["prescriptions", "inventory", "stock_management"],
    "staff": ["patient_management", "appointments"],
    "cashier": ["billing", "payments", "basic_patient_info"]
  },
  "testScenarios": [
    {
      "action": "create_user",
      "user": "admin",
      "targetRole": "doctor"
    },
    {
      "action": "modify_permissions",
      "user": "admin", 
      "targetUser": "staff01",
      "newRole": "pharmacist"
    },
    {
      "action": "suspend_user",
      "user": "admin",
      "targetUser": "inactive_user"
    }
  ]
}
```

**การตรวจสอบ**:
- [ ] Admin สามารถสร้างผู้ใช้ใหม่ได้
- [ ] ระบบกำหนดบทบาทและสิทธิ์ได้ถูกต้อง
- [ ] การเปลี่ยนแปลงสิทธิ์มีผลทันที
- [ ] ผู้ใช้ถูกจำกัดการเข้าถึงตามบทบาท
- [ ] Admin สามารถระงับบัญชีได้
- [ ] ผู้ใช้ที่ถูกระงับไม่สามารถเข้าสู่ระบบได้
- [ ] ระบบบันทึกการเปลี่ยนแปลงสิทธิ์

### UAT-ID: UAT-006
**บทบาท/Use Case**: รายงานและวิเคราะห์
**ขั้นตอน**:
1. เปิดรายงานรายเดือน
2. สรุปยอดขายยา

**เกณฑ์ยอมรับ**: ตัวเลขตรง DB, ส่งออกไฟล์ (CSV/PDF) ได้

**ข้อมูลทดสอบ**:
```json
{
  "reportPeriod": {
    "start_date": "2024-01-01",
    "end_date": "2024-01-31"
  },
  "expectedReports": [
    {
      "type": "monthly_sales",
      "content": ["total_revenue", "transaction_count", "top_medications"]
    },
    {
      "type": "patient_statistics", 
      "content": ["new_patients", "returning_patients", "age_distribution"]
    },
    {
      "type": "inventory_report",
      "content": ["stock_levels", "low_stock_alerts", "expiry_warnings"]
    }
  ],
  "exportFormats": ["CSV", "PDF", "Excel"],
  "dataAccuracy": "100% match with database"
}
```

**การตรวจสอบ**:
- [ ] รายงานรายเดือนแสดงข้อมูลครบถ้วน
- [ ] ตัวเลขในรายงานตรงกับฐานข้อมูล
- [ ] รายงานสรุปยอดขายยาถูกต้อง
- [ ] สามารถกรองข้อมูลตามช่วงเวลาได้
- [ ] สามารถส่งออกเป็น CSV ได้
- [ ] สามารถส่งออกเป็น PDF ได้
- [ ] สามารถส่งออกเป็น Excel ได้
- [ ] ไฟล์ที่ส่งออกมีข้อมูลครบถ้วน

### UAT-ID: UAT-007
**บทบาท/Use Case**: ความปลอดภัย/การเข้าถึง
**ขั้นตอน**: ผู้ใช้บทบาทต่ำลองเข้าถึงข้อมูลจำกัด

**เกณฑ์ยอมรับ**: ระบบปฏิเสธ (403), บันทึก Audit

**ข้อมูลทดสอบ**:
```json
{
  "unauthorizedAccess": [
    {
      "user": "cashier01",
      "attemptedAccess": "medical_records",
      "expectedResult": "403 Forbidden"
    },
    {
      "user": "pharmacist01", 
      "attemptedAccess": "financial_reports",
      "expectedResult": "403 Forbidden"
    },
    {
      "user": "staff01",
      "attemptedAccess": "user_management",
      "expectedResult": "403 Forbidden"
    }
  ],
  "auditLogging": {
    "requiredFields": ["user", "action", "resource", "timestamp", "ip_address", "result"],
    "retentionPeriod": "1 year"
  }
}
```

**การตรวจสอบ**:
- [ ] เจ้าหน้าที่การเงินไม่สามารถเข้าถึงเวชระเบียนได้
- [ ] เภสัชกรไม่สามารถเข้าถึงรายงานการเงินได้
- [ ] เจ้าหน้าที่ไม่สามารถจัดการผู้ใช้ได้
- [ ] ระบบส่งคืน HTTP 403 Forbidden
- [ ] ระบบแสดงข้อความ "ไม่มีสิทธิ์เข้าถึง"
- [ ] การพยายามเข้าถึงถูกบันทึกใน Audit Log
- [ ] Audit Log มีข้อมูลครบถ้วน (user, action, resource, time, IP, result)
- [ ] ไม่มีการรั่วไหลของข้อมูล

## เกณฑ์การยอมรับโดยรวม

### 1. ระบบจัดการข้อมูลผู้ป่วย (1.3.1)
- [ ] สามารถบันทึกและจัดเก็บข้อมูลผู้ป่วยได้ (UAT-001)
- [ ] สามารถเรียกดูข้อมูลผู้ป่วยได้ (UAT-001)

### 2. ระบบเวชระเบียนและการรักษา (1.3.2)
- [ ] สามารถบันทึกข้อมูลการรักษาได้ (UAT-002)
- [ ] สามารถจัดการข้อมูลยาของผู้ป่วยได้ (UAT-002, UAT-003)

### 3. ระบบคลังยาและสต๊อกยา (1.3.3)
- [ ] สามารถบริหารจัดการสต๊อกยาได้ (UAT-003)
- [ ] สามารถบันทึกและสั่งซื้อยาได้ (UAT-003)

### 4. ระบบคิดเงินและออกใบเสร็จ (1.3.4)
- [ ] สามารถคำนวณค่ารักษาได้ (UAT-004)
- [ ] สามารถออกใบเสร็จและบันทึกข้อมูลการเงินได้ (UAT-004)

### 5. ระบบสำรองข้อมูลและความปลอดภัย (1.3.5)
- [ ] สามารถสำรองข้อมูลได้ (IT-005)
- [ ] สามารถรักษาความปลอดภัยของข้อมูลได้ (UAT-007)

### 6. ระบบจัดการผู้ใช้งานและสิทธิ์การเข้าถึง (1.3.6)
- [ ] สามารถสมัครสมาชิกและเข้าสู่ระบบได้ (UAT-001)
- [ ] สามารถกำหนดสิทธิ์ของผู้ใช้ได้ (UAT-005)

### 7. ระบบรายงานและวิเคราะห์ข้อมูล (1.3.7)
- [ ] สามารถสร้างรายงานข้อมูลได้ (UAT-006)
- [ ] สามารถวิเคราะห์แนวโน้มของข้อมูลได้ (UAT-006)

## หมายเหตุ
- การทดสอบ UAT ควรทำโดยผู้ใช้งานจริงในสภาพแวดล้อมจริง
- ควรเก็บข้อมูล feedback จากผู้ใช้งานเพื่อปรับปรุงระบบ
- การทดสอบควรครอบคลุมทั้งกรณีปกติและกรณีขอบเขต
- ควรทดสอบประสิทธิภาพและความสะดวกในการใช้งาน
