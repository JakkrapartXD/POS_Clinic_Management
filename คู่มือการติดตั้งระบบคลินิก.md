# คู่มือการติดตั้งระบบคลินิก (Clinic System Installation Guide)

## 📋 ภาพรวมระบบ

ระบบจัดการคลินิกแบบครบวงจรที่ประกอบด้วย:
- **ระบบ POS** - จัดการการขายและชำระเงิน
- **ระบบคลินิก** - จัดการผู้ป่วย การตรวจ และคิว
- **ระบบจัดการสินค้า** - จัดการสต็อกและหมวดหมู่
- **ระบบรายงาน** - สถิติและการวิเคราะห์
- **ระบบสำรองข้อมูล** - สำรองข้อมูลอัตโนมัติ

## 🛠️ เทคโนโลยีที่ใช้

- **Frontend**: Next.js 15, React 19, TypeScript, Tailwind CSS
- **Backend**: Elysia.js, GraphQL, Prisma ORM
- **Database**: PostgreSQL 16
- **Cache**: Redis 7
- **Container**: Docker & Docker Compose

## 📦 ขั้นตอนที่ 1: ติดตั้ง Docker และ Docker Compose

### สำหรับ macOS

#### วิธีที่ 1: ติดตั้งผ่าน Docker Desktop (แนะนำ)
1. ไปที่ [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/)
2. ดาวน์โหลดไฟล์ `.dmg`
3. เปิดไฟล์และลาก Docker ไปยัง Applications folder
4. เปิด Docker Desktop จาก Applications
5. รอให้ Docker เริ่มทำงาน (จะเห็นไอคอน Docker ในเมนูบาร์)

#### วิธีที่ 2: ติดตั้งผ่าน Homebrew
```bash
# ติดตั้ง Homebrew (ถ้ายังไม่มี)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# ติดตั้ง Docker
brew install --cask docker

# เปิด Docker Desktop
open /Applications/Docker.app
```

### สำหรับ Windows

1. ไปที่ [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
2. ดาวน์โหลดไฟล์ `.exe`
3. รันไฟล์ติดตั้งและทำตามขั้นตอน
4. รีสตาร์ทคอมพิวเตอร์เมื่อเสร็จสิ้น
5. เปิด Docker Desktop

### สำหรับ Linux (Ubuntu/Debian)

```bash
# อัปเดตแพ็กเกจ
sudo apt update

# ติดตั้ง dependencies
sudo apt install apt-transport-https ca-certificates curl gnupg lsb-release

# เพิ่ม Docker GPG key
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# เพิ่ม Docker repository
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# ติดตั้ง Docker
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-compose-plugin

# เพิ่มผู้ใช้เข้า Docker group
sudo usermod -aG docker $USER

# รีสตาร์ทเซสชันหรือรันคำสั่งนี้
newgrp docker
```

### ตรวจสอบการติดตั้ง

```bash
# ตรวจสอบ Docker
docker --version

# ตรวจสอบ Docker Compose
docker compose version

# ทดสอบการทำงาน
docker run hello-world
```

## ⚙️ ขั้นตอนที่ 3: ตั้งค่า Environment Variables

### 1. สร้างไฟล์ Environment สำหรับ Root Directory

```bash
# สร้างไฟล์ .env สำหรับ Docker services
touch .env
```

**เนื้อหาไฟล์ `.env`:**
```env
# Database Configuration
POSTGRES_DB=sn_clinic
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your_secure_password_here
PGADMIN_DEFAULT_EMAIL=admin@clinic.local
PGADMIN_DEFAULT_PASSWORD=admin1234
```

### 2. สร้างไฟล์ Environment สำหรับ Development

```bash
# สร้างไฟล์ .env.development
touch .env.development
```

**เนื้อหาไฟล์ `.env.development`:**
```env
# Application Environment
NODE_ENV=development

# JWT Configuration
JWT_SECRET=your-super-secret-jwt-key-for-development-only
JWT_EXPIRES_IN=24h

# Database Configuration
DATABASE_URL=postgresql://postgres:your_secure_password_here@postgres:5432/sn_clinic

# Redis Configuration
REDIS_URL=redis://redis:6379

# Frontend Configuration
FRONTEND_URL=http://localhost:3000

# Admin User Configuration
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin1234
```

### 3. สร้างไฟล์ Environment สำหรับ Backend

```bash
# สร้างไฟล์ .env ใน backend directory
touch backend/.env
```

**เนื้อหาไฟล์ `backend/.env`:**
```env
# JWT Configuration
JWT_SECRET=your-super-secret-jwt-key-for-development-only
JWT_EXPIRES_IN=24h

# Database Configuration
DATABASE_URL=postgresql://postgres:your_secure_password_here@postgres:5432/sn_clinic

# Redis Configuration
REDIS_URL=redis://redis:6379

# Frontend Configuration
FRONTEND_URL=http://localhost:3000

# Admin User Configuration
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin1234

# Application Configuration
NODE_ENV=development
PORT=4000
```

### 4. ตั้งค่าความปลอดภัย

```bash
# เปลี่ยนสิทธิ์ไฟล์ .env ให้ปลอดภัย
chmod 600 .env
chmod 600 .env.development
chmod 600 backend/.env
```

### 1. ตั้งค่า Google Drive (สำหรับการสำรองข้อมูล)

   - **Host**: postgres

## 🚨 การแก้ไขปัญหาที่พบบ่อย

### 1. Docker ไม่เริ่มทำงาน

**ปัญหา**: Docker Desktop ไม่เปิดหรือ containers ไม่รัน

**วิธีแก้**:
```bash
# ตรวจสอบ Docker daemon
docker info

# รีสตาร์ท Docker Desktop
# macOS: เปิด Activity Monitor และปิด Docker Desktop แล้วเปิดใหม่
# Windows: รีสตาร์ท Docker Desktop จาก system tray
# Linux: sudo systemctl restart docker
```

### 2. Port ถูกใช้งานแล้ว

**ปัญหา**: Port 3000, 4000, 5432, หรือ 6379 ถูกใช้งานแล้ว

**วิธีแก้**:
```bash
# ตรวจสอบ port ที่ใช้งาน
lsof -i :3000
lsof -i :4000
lsof -i :5432
lsof -i :6379

# ปิด process ที่ใช้ port
kill -9 <PID>

# หรือเปลี่ยน port ใน docker-compose.yml
```

### 3. ฐานข้อมูลเชื่อมต่อไม่ได้

**ปัญหา**: Backend ไม่สามารถเชื่อมต่อฐานข้อมูลได้

**วิธีแก้**:
```bash
# ตรวจสอบ postgres container
docker compose logs postgres

# รีสตาร์ท postgres
docker compose restart postgres

# ตรวจสอบ network
docker network ls
docker network inspect final_project_app-network
```

### 4. Migration ล้มเหลว

**ปัญหา**: Prisma migration ไม่สำเร็จ

**วิธีแก้**:
```bash
# เข้าไปใน backend container
docker compose exec backend bash

# รีเซ็ตฐานข้อมูล
npx prisma migrate reset

# รัน migration ใหม่
npx prisma migrate deploy

# สร้าง seed data ใหม่
npx prisma db seed
```

### 5. Frontend ไม่โหลด

**ปัญหา**: Frontend ไม่แสดงผลหรือแสดง error

**วิธีแก้**:
```bash
# ตรวจสอบ frontend logs
docker compose logs frontend

# รีสตาร์ท frontend
docker compose restart frontend

# ตรวจสอบ environment variables
docker compose exec frontend env | grep NODE_ENV
```

### 6. JWT Token ไม่ถูกต้อง

**ปัญหา**: Authentication ล้มเหลว

**วิธีแก้**:
```bash
# ตรวจสอบ JWT_SECRET ใน .env
grep JWT_SECRET backend/.env

# สร้าง JWT secret ใหม่
openssl rand -base64 32

# อัปเดต .env และรีสตาร์ท services
docker compose restart backend frontend
```

### 7. Redis Connection Error

**ปัญหา**: Redis เชื่อมต่อไม่ได้

**วิธีแก้**:
```bash
# ตรวจสอบ redis container
docker compose logs redis

# ทดสอบ redis connection
docker compose exec redis redis-cli ping

# รีสตาร์ท redis
docker compose restart redis
```

## 🗂️ การสำรองและกู้คืนข้อมูล

### 1. สำรองข้อมูล

```bash
# สำรองฐานข้อมูล
docker compose exec postgres pg_dump -U postgres sn_clinic > backup_$(date +%Y%m%d_%H%M%S).sql

# สำรองไฟล์ uploads
docker compose exec backend tar -czf uploads_backup_$(date +%Y%m%d_%H%M%S).tar.gz /app/public/uploads/
```

### 2. กู้คืนข้อมูล

```bash
# กู้คืนฐานข้อมูล
docker compose exec -T postgres psql -U postgres sn_clinic < backup_file.sql

# กู้คืนไฟล์ uploads
docker compose exec backend tar -xzf uploads_backup_file.tar.gz -C /
```
